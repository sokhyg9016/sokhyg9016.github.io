다양한 예제로 시작하는 포인터

작성일 2020-06-14 Edited on 2020-06-17 In Programming , C

자료형과 변수, 상수에 대한 정리

  • 자료형: 일정 길이의 메모리에 저장되어 있는 데이터를 해석하는 방법, 또는 형식을 말한다. (즉, 메모리를 사용할 때 몇 바이트의 메모리를 사용할 것인지를 명시해주는 것이다.) [메모리를 표현하는 단위]
  • 변수: 정해지지 않은 수/ 변할 수 있는 수, 일정 길이의 메모리에 저장되어 있는 데이터가 있을 때, 그 특정 메모리 공간에 붙여진 이름(또는 공간 자체)을 말한다.
  • 상수: 값이 정해진수, 프로그램에서 값이 변경되지 않는 요소이다. (e.g.s 리터럴 상수, 이름이 있는 매크로 상수, const 지정 변수 등)

시작주소란?

모든 변수는 메모리 공간의 주소를 가지고 있고, 이때 그 변수를 대표하는 주소를 시작 주소라고 한다.

컴퓨터의 메모리

  • 메모리는 데이터를 저장할 수 있는 저장 장치이다.
  • 주 기억 장치 (RAM/ROM), 보조 기억 장치 (HDD, FDD, CD-ROM) 등을 말한다.
  • 프로그램에서는 데이터의 저장을 주로 주 기억 장치인 RAM에서 하게 되고 보통 프로그램에서 메모리라고 하면 RAM의 기억 공간을 의미한다.
  • 메모리의 주소는 보통 1바이트의 저장 공간을 가지며, 1바이트는 8비트이다.
  • 주소 값의 범위는 OS에 따라 차이가 있는데, 32-bit OS에서는 32 비트를 넘지 않으며, 64-bit OS에서는 64비트를 넘지 않는다.
  • 모든 변수는 메모리 공간에 할당되고 변수의 크기는 자료형으로 결정된다. (따라서 자료형을 보면 변수의 크기와 메모리 주소의 사용 개수를 알 수 있다.)

변수의 주소

변수들의 주소는 자료형에 따라 주소의 의미가 다르다. 예를 들어 변수의 주소에 정수를 덧셈하면 변수의 자료형의 크기만큼 메모리 공간을 건너뛴 다음 메모리의 주소를 구한다.

즉, char형 주소는 1바이트에 데이터를 가지고 있는 시작 주소이고, int형 주소는 4바이트 데이터를 가지고 있는 시작 주소이기 때문에 가지고 있는 데이터만큼 건너 뛰게 된다.

주소의 형 변환

형 변환은 암묵적인 형 변환과 명시적인 형 변환이 있다.

암묵적 형 변환 명시적 형 변환
컴파일러에 의해 자동 변환 () 연산자(casting)를 사용해 변환

작은 자료형을 큰 자료형으로 변환하는 경우 컴파일러는 자동으로 형 변환을 수행한다. (컴파일러가 내부적으로 형 변환)

또한 변수의 주소도 자료 형식이 존재한다. 따라서 주소도 변수의 자료형과 마찬가지로 명시적인 형 변환이 가능하다.
