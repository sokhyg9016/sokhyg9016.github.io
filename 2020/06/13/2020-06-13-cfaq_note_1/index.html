Q 2.8 왜 == 나 !=를 써서 구조체를 비교할 수 없나요?

C언어의 저수준성을 고려하여, 구조체 비교를 구현하기 위한 좋은 방법은 컴파일러 입장에서는 존재하지 않습니다.

이는 구조체 필드 사이에 있는 hole들을 생각할 때 좋지 않기 때문입니다. 이런 점들은 각 필드에 주어진 정렬 (alignment) 방법에 맞도록 위치시키는 것이 중요합니다.

6.2 char a[]는 char* a와 같지 않나요?

전혀 다릅니다. (여러분이 생각한 것은 아마도 함수의 formal parameter에 관한 것일 겁니다. 질문 6.4를 참고하기 바랍니다.) 비록 서로 연관되어 있고 또 비슷하게 쓰이기도 하지만, 배열은 포인터가 아닙니다.

아래 코드가 주어졌을 때,

  • char a[6]과 같은 선언은 문자 여섯 개를 저장할 수 있는 공간을 요청하고, 그 결과 a라는 이름이 이 공간을 대표하게 됩니다.
  • 반면 포인터 선언 char *p는 포인터를 저장할 수 있는 공간을 요구하고 이 위치는 p라고 이름 지어집니다.
  • 이 포인터는 이제 어떤 곳도 가리킬 수 있습니다. (정확히 말해 어떤 문자나, 문자로 이루어진 배열의 한 요소를 가리킬 수도 있으며, 아무것도 가리키지 않을 수도 있습니다.)

이는 다음과 같이 데이터를 초기화 합니다.

위 두 형식에서 x[3]과 같은 식은 서로 다른 코드를 생성합니다.

statement description
a[3] 주소 a에서 3개 만큼 지나서 그 위치에 문자를 가져온다.
p[3] 주소 p에서 3을 더한 다음 그 위치에 문자를 가져온다.

결과 값은 같지만 컴파일러는 이곳을 찾기 위해 다른 코드를 사용합니다.

가장 큰 차이는 a와 같은 배열과 p와 같은 포인터의 값을 계산할 때 컴파일러가 서로 다른 방법을 써서 계산한다는 것입니다. 이 계산은 subscript 연산자 []를 쓰는지의 여부와 관계없습니다.

Q 1.32 다음 두개의 차이는 무엇인가요?

  • 1번의 경우는 배열의 초기값으로 쓰이는 경우로, 배열의 각 요소들인 문자들에 대입되는 초기값을 나타낸다.
  • 2번의 경우는 문자열이 이름이 있는 정적인 배열에 저장되고 – 대개 이 배열은 읽기 전용 (Read-Only)의 속성을 가진다. – 수식에서 쓰일 때는 이 배열의 1번째 요소를 가리키는 포인터로서 쓰인다.

Q 6.3 “포인터와 배열이 같다” 라는 말의 의미?

이 말의 의미는 포인터와 배열이 서로 같다(identical)의 의미도 아니고, 서로 마음대로 바꿔서 쓸 수 있다. (interchangeable)의 의미도 아닙니다.

이에 관하여 다음과 같이 정리할 수 있습니다.

An lvalue of type array-of-T which appears in an expression decays (with three exceptions) into a pointer to its ﬁrst element; the type of the resultant pointer is pointer-to-T. (The exceptions are when the array is the operand of a sizeof or an & operator or is a string literal initializer for a character array.)

이말의 의미는 즉, T타입의 배열의 L-Value가 수식에서 나타날 때에는 배열의 1번째 요소를 가리키는 포인터로 변경(decay)된다는 것입니다.

That is, whenever an array appears in an expression, the compiler implicitly generates a pointer to the array’s first element, just as if the programmer had written &a[0].

따라서 배열의 식별자는 배열의 주소나 첫번째 요소 뿐만 아니라 배열 자체를 식별하는데도 사용됩니다. 아래 3가지 예외의 경우에서 확인할 수 있습니다.

  • 배열이 sizeof의 피연산자로 쓰일 때
  • 배열이 & 연산자의 피연산자로 쓰일때
  • 문자 배열에서 초기값인 문자열로 쓰일 때

위의 경우 먼저 sizeof(arr) 와 같이 쓰일 경우, (arr: 배열 이름) 배열 이름이 포인터로 변경되는 것이 아닌 해당 배열의 크기로 변경되어 계산됩니다. 또한 단항 연산자 &의 경우 피연산자의 주소를 산출하며 이때 또한 포인터로 변경되지 않습니다.

3 Except when it is the operand of the sizeof operator or the unary & operator, or is a string literal used to initialize an array, an expression that has type “array of type” is converted to an expression with type ‘‘pointer to type’’ that points to the initial element of the array object and is not an lvalue. If the array object has register storage class, the behavior is undefined.

참고: C Standard (6.3.2 Other operands) - p.72

이때 변경된 포인터의 타입은 T타입을 가리키는 포인터(Pointer to type T)가 됩니다.

이 정의에 따라서 내부적으로 배열이 포인터와는 매우 다르지만, 배열이냐 포인터냐에 상관없이 [](subscript operator)를 사용할 수 있습니다. 예를 들어,

$$ p = a; $$

이때 p는 포인터로 퇴화된(decay) a의 값이 대입됩니다.

이렇기 때문에 포인터를 써서 배열에 접근할 수 있고 함수 parameter로 쓸 수 있으며, dynamic array를 흉내낼 수 있습니다. (참고 -> 6.4, 6.14)
